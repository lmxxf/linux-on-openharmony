# RK3588 欧拉 Linux 桌面浏览器安装指南

在没有桌面环境的欧拉 Linux（RK3588）上，通过 VNC + Firefox 实现远程桌面浏览器。

## 安装

```bash
# VNC 服务 + Firefox 浏览器
dnf install tigervnc-server firefox

# 中文字体（否则中文显示为方块）
dnf install google-noto-sans-cjk-ttc-fonts
```

## 配置

```bash
# 设置 VNC 密码
vncpasswd

# 配置 VNC 启动时直接打开 Firefox
mkdir -p ~/.vnc
cat > ~/.vnc/xstartup << 'EOF'
#!/bin/sh
firefox &
EOF
chmod +x ~/.vnc/xstartup
```

## 启动

```bash
vncserver :1 -geometry 1920x1080 -localhost no
```

## 连接

PC 端安装 VNC 客户端（RealVNC Viewer / TightVNC），连接 `3588的IP:5901`。

## 常用操作

```bash
# 停止
vncserver -kill :1

# 重启（改配置后需要）
vncserver -kill :1
vncserver :1 -geometry 1920x1080 -localhost no

# 改分辨率
vncserver -kill :1
vncserver :1 -geometry 2560x1440 -localhost no
```

## VSCode Remote SSH 连接

VSCode Remote SSH 需要 TCP 端口转发，欧拉默认可能禁用了。如果连接报错 `TCP port forwarding appears to be disabled`：

```bash
# 检查当前配置
grep -i AllowTcpForwarding /etc/ssh/sshd_config

# 如果是 no，改成 yes
sed -i 's/AllowTcpForwarding no/AllowTcpForwarding yes/' /etc/ssh/sshd_config

# 如果没有这行，直接加上
echo "AllowTcpForwarding yes" >> /etc/ssh/sshd_config

# 重启 sshd
systemctl restart sshd
```

> 注意：删除 `~/.vscode-server` 不能解决此问题，必须改 sshd 配置。

## 说明

- 欧拉仓库没有 XFCE/GNOME/OpenBox 等桌面环境和窗口管理器，所以 VNC 连上去没有桌面、没有任务栏，直接就是全屏 Firefox
- 如果 Firefox 意外关闭，VNC 会变成黑屏，需要 kill 重启
- Kimi Code CLI 等需要浏览器授权登录的工具，也可以不装桌面，直接把终端里显示的授权 URL 复制到任意设备的浏览器上打开即可（Device Authorization Flow，轮询机制，不需要本地浏览器）
